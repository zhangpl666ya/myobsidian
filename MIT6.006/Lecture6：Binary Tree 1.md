# 1️⃣二叉树
目前为止最为优雅、综合效率最高的数据结构
## 结构实现
- 基于指针的数据结构（像链表），能实现最差情况的表现
- 二叉树的节点由三个指针
- 节点表现为：`node.{item,parent,left,right}`

## 术语
- **根**是指没有父节点的节点
- **叶**是没有子节点（left、right）的节点
- 定义`depth(<x>)`是x节点从根节点到该节点的边的数量
- 定义`height(<x>)`是x节点到最远叶节点的边的数量
- **想法**：设计操作时间在O（h），h是根节点的高度，同时维护h = O（logn）
- 一个二叉树有内在顺序：中序遍历：
	- 每个节点的左子树的节点都在该节点左边
	- 每个节点的右子树的节点都在该节点右边
- 连接的节点在中序遍历从根节点开始：
	- 递归的列出左子树，列出自身，递归地列出右子树
	- 时间复杂度在O（n）

## 树的导航
- **find first**：找出以x为根节点的子树中序遍历中第一个节点名（find last 对称）
	- 若x有左节点，那么递归的返回左子树的第一个节点
	- 不然，x就是要找到节点，返回x
	- 时间在O（h），h是树的高度
- **Find successor**：找出中序遍历下x的下一个节点（前驱节点对称）
	- 若x有右节点，返回右子树的第一个节点
	- 不然，找到第一个祖先节点，x在这个祖先节点的左子树中
	- 时间在O（h），h是树的高度

## 动态操作
- 改变树的一个节点（只添加或移除叶节点）
	- 在另一个节点前（或者后）添加一个节点
	- 从树中移除一个节点
- **Insert**：中序遍历下在x节点后插入节点Y
	- 若x没有右子树，那么Y直接成为x的右节点
	- 不然，让Y成为x节点的前驱节点的左节点
- **Delete**：删除以x为根节点的树中的节点
	- 如果x是叶节点，那么直接将x分离
	- 不然，若有左子树，将x与其前驱节点递归的交换，直至x成为叶节点
	- 再不然，x有右子树，将x递归的与右子树交换，直至x成为叶节点
	- 时间在O（n）


# 2️⃣应用
## Set
- **想法**：结合set和二叉树（亦称二叉搜索树/BST）
	- 拥有BST性质：每一个节点，他的左子树的所有节点值都小于自身，右子树所有节点的值大于自身
- 用键值找以x为根节点的节点，就像二分查找：
	- 若k比x小，就递归x的左子树
	- 若比x大，就递归右子树
	- x=k，找到

## Sequence
- **想法**：树的遍历顺序就是序列顺序
- 我们怎么找到索引为i的节点？调用函数`subtree_at(i)`
- 可以迭代整个顺序，但时间复杂度高
- 不过我们若可以在常数时间内计算出子树的大小，那么可以做到O（n）
	- 检查根节点左子树包含的节点数n
	- 若 i < n：递归根节点的左节点
	- 若 i > n：递归根节点的右节点，i‘ = i - n - 1
	- i = n 找到
- 通过在节点中添加`node.size`记录子树大小
	- 添加节点时，给所有祖先节点的`.size`加一
	- 删除节点时，给所有祖先节点的`.size`减一
