# 二进制模式模式与文本模式的区别

## 文本模式
python给你用编码协议解释内容
```python
with open("test.txt","r") as f:
	data = f.read()
```
这不是“原始读取”，而是：

1. 从磁盘读 **字节**
    
2. 按 `encoding` 解码成 **字符**
    
3. 可能做 **换行符转换**（`\r\n` → `\n`）
    
4. 返回 `str`
    

📌 所以你拿到的是：  
**“已经被 Python 加工过的文本”**

## 二进制模式
python只搬运字节，不理解内容，什么都不加工
```python
with open("test.txt","rb") as f:
	data = f.read()
	
# 你得到的是：
#<class 'bytes'>
b'Hello\nWorld'   #前面加b表示binary
```
📌 关键点：

- 没有编码
    
- 没有换行转换
    
- 一个字节都不改
	
- 一个字节 = 一个单位
👉 **这和 C/C++ 的 `fread` 思维是完全一致的**


| 维度       | 文本模式  | 二进制模式   |
| -------- | ----- | ------- |
| 返回类型     | `str` | `bytes` |
| 编码       | 有     | 无       |
| 换行处理     | 有     | 无       |
| seek 精确  | ❌     | ✅       |
| 接近 C/C++ | ❌     | ✅       |

# 什么时候一定要用二进制模式
你可以记这张“工程判断表”👇

|场景|模式|
|---|---|
|文本（配置、源码、日志）|`"r"`|
|图片 / 音频 / 视频|`"rb"`|
|压缩包 / exe / pdf|`"rb"`|
|网络协议 / 自定义格式|`"rb"`|
|精确 `seek`|`"rb"`|

📌 **凡是你“不关心字符含义”的，直接二进制**
