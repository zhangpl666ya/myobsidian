不论是什么编程语言，文件的读写从来不是从头到尾，而是有个指针的，以指示目前从哪里开始读/写

>系统内部维护着一个“偏移量”（offset）

# 指针现在在哪？

`tell()`

```python
with open("test.txt","r") as f:
	print(f.tell())
```

返回的是当前指针距离文件开头的**字节数**

# 指针怎么动？

[[stdio.h#文件定位|C语言看这里]]

```python
with open("test.txt","r") as f:
	print(f.tell()) #0
	f.read('5')
	print(f.tell()) #5
```

- 文件打开：指针在0
- read（5）：读了5个字节
- 指针移动5个字节
- 读操作：移动指针

## `seek()`
>`seek()`用来“手动移动文件指针”

最基础的用法：

`f.seek(offset)`

```python
with open("test.txt","r") as f:
	f.read(5)
	f.seek(0)
	print(f.read(5))
```

👉读了5个字节后跳回文件开头

完整用法：`f.seek(offset,whence)`

- `whence`含义：
whence                                  含义
---
0                                            从文件开头（默认）

1                                            从当前位置

2                                            从文件末尾

---

## 常见现象：
为什么读完一次，再读文件就没有内容了？
```python
with open("test.txt",'r') as f:
	f.read()
	print(f.read())   #空
```
👆**因为文件指针已经到文件末尾了**

加一行：`f.seek(0)`即可

>[!warning]
>文本模式下使用seek函数非常危险
>因为编码的情况下一个字符可能对应1~4个字节，字节偏移很容易跳到字符中间
>推荐在[[二进制模式]]下使用
