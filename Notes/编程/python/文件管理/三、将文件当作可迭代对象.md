## 把文件当成「可迭代对象」来读

👉 `for line in f`

## 最典型的写法（你以后会常见到）

```python
with open("test.txt","r") as f:
	for line in f:
		print(line)
```

不新鲜，但**意义非常大**。

### 这和`readlines()`有什么本质区别？
我们知道：
```python
lines = f.readlines()
# 返回列表，元素为每一行的内容
```
这是：**一次性将所有行读进内存**
而：
`for line in f`
是 **一行一行地“流式读取”**
### 🔥 关键区别在这里

|方式|内存占用|适合场景|
|---|---|---|
|`read()`|最大|小文件|
|`readlines()`|大|中小文件|
|`for line in f`|**最小**|**大文件 / 日志**|
（相当于[[Generator Expressions]]，需要时生成）

📌 所以 **工程里默认推荐 `for line in f`**

### 为什么文件可以被for遍历

***因为文件对象实现了“迭代器协议”***

内部相当于：
```python
while True:
	line = f.readline()
	if line == "":
		break
	yield line
```

[[Yield关键字]]看这里

小坑：
	python 中的print是自带换行的
	但是若文件自己也有换行符，就会出现小问题
	所以正确写法：`print(line.rstrip())`
	或：
	`print(line,end = "")`

## 典型使用场景示例

### 场景：逐行处理文件内容

`with open("data.txt") as f:     for line in f:         if "error" in line:             print(line)`

📌 你**不需要关心文件有多大**  
📌 Python 帮你一行一行喂

## 这一种方法的定位

✅ 优点：

- 最省内存
    
- 最 Pythonic
    
- 工程首选
    
- 可读性极强
    

❌ 缺点：

- 不适合需要随机访问（比如第 100 行）


## 与writelines配合：

### `writelines()` 到底干了什么？

一句话：

> **把一个「字符串可迭代对象」里的每个元素，依次写入文件**

⚠️ 重点注意（很多人第一次会踩）：

### ❌ `writelines()` **不会自动加换行符**

```python
lines = ["a", "b", "c"]
f.writelines(lines)
```

结果是：

`abc`

所以你**必须自己带 `\n`**。

## 最常见的真实用法（🔥）

### 场景：边处理边生成输出文件

```python
with open("input.txt") as fin, open("output.txt","w") as fout
	fout.writelines(
		line.upper() for line in fin
	)
```
