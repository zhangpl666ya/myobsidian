# 最基础、最原始的方式——`open/read/write/close`

这是文件操作的地基，
后面的“高级写法”底层都绕不开

## 一、文件操作核心思想
1. **打开文件**：得到一个**文件对象**
2. **对文件对象读/写**
3. **关闭文件**：把资源还给系统

## 二、打开文件：`open()'

`f = open("test.txt","w")`
`open(文件名，模式）`

常见模式：（这个跟[[stdio.h#文件管理：打开、关闭和删改文件|C语言的]]很像）

| 模式   | 含义               |
| ---- | ---------------- |
| “r"  | 只读（文件必须存在）       |
| "w"  | 只写（**清空原文件或新建**） |
| ”a"  | 追加写              |
| “r+" | 读+写              |

## 三、写文件：`write()`

```python
f = open("test.txt","w")
f.write("Hello Python\n")
f.write("第二行")
f.close()
```

>[!注意]
>1. `write()` **不会自动换行**
>2. 写入的是 **字符串**
>3. 不写`close()`，数据可能不会真正写入磁盘

## 四、读文件：`read()`
1. 一次性读完
```python
f= open("test.txt", "r")
content = f.read()
print(content)
f.close()
```
2. 读一行
```python
f = open("test.txt","r")
line = f.readline()
print(line)
f.close()
```
3. 读所有行
```python
f = open("test.txt", "r")
lines = f.readlines()
print(lines)
f.close()

# 结果：['Hello Python\n', '第二行']
```

## 五、为什么一定要 `close()`

不关文件会带来：

- 文件句柄泄露
    
- Windows 下文件被锁，无法删除
    
- 写入内容不完整
    

所以：  
**这一版写法是“能用但不优雅”**

## 六、这一种方法的定位

✅ 优点：

- 最直观
    
- 好理解
    
- 适合理解底层机制

❌ 缺点：

- 容易忘 `close()`
    
- 出异常时不会自动关闭
