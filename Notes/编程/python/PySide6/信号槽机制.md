# 信号槽（Signal & Slot）— PySide6

## 什么是信号槽

**信号（Signal）** — 对象 `emit` 出去的事件通知。按钮被点了、输入框内容变了、窗口要关闭了——这些都是信号。

**槽（Slot）** — 接收信号并执行反应的函数/方法。Signal fires → slot is called. The sender doesn't care who receives it. The receiver doesn't care who fires it. **Decouple** event emitter and event handler. `connect()` wires them, `emit()` fires them all.

## 内置信号 — `connect()` / `disconnect()`

### 常用内置信号速查

| 信号 | 触发时机 | 携带参数 |
|------|----------|----------|
| `clicked` | 按钮点击 | bool checked=False |
| `pressed` | 按钮按下 | — |
| `released` | 按钮释放 | — |
| `toggled(bool)` | 复选框/可点按钮状态切换 | bool checked |
| `checked` | 同 `toggled`，仅 CheckBox | bool |
| `textChanged(str)` | 行编辑器文字变化 | str — 变化后的新文本 |
| `currentIndexChanged(int)` | 组合框选中项变化 | int index |
| `currentTextChanged(str)` | 组合框选中文字变化 | str text |
| `valueChanged(int)` | spinbox / slider 数值变化 | int |

### `connect()` — 连接信号与槽

```python
# 1. 基本连接（slot 参数自动和 signal 参数匹配）
btn.clicked.connect(self.handle_click)

def handle_click(self, checked: bool):  # checked 可省略，不接收也行
    print('按钮被点了')

# 2. lambda 包装（过滤或额外处理）
combo.currentIndexChanged.connect(lambda idx: self.on_changed(index=idx))
slider.valueChanged.connect(lambda val: self.on_slider(val * 2))
```

### `disconnect()` 断开连接

```python
btn.clicked.disconnect(self.handle_click)   # 断开特定槽
combo.currentIndexChanged.disconnect()       # 断开所有槽（清空整个连接列表）

# 主动断开在生命周期管理时很重要——widget 销毁前断开 signal 是好的清理习惯：
self.destroyed.connect(self.on_widget_destroyed)
```

## 自定义信号 `Signal`

```python
from PySide6.QtCore import Signal

class MyWidget(QWidget):
    # 在类属性位置用 Signal() 定义自定义信号
    count_changed = Signal()                  # 无参信号
    count_changed_int = Signal(int)           # 带一个 int 参数
    multi_signal = Signal(int, str)           # 两个参数：int 和 str

    def __init__(self):
        super().__init__()
        self.count_changed_int.connect(self.on_count_changed)

    def on_count_changed(self, value: int):
        print(f'计数: {value}')

    def increment(self):
        self.count_changed_int.emit(42)  # 发射信号，int 参数传给槽
        self.multi_signal.emit(42, 'hello')  # 多参数按顺序对应
```

### Signal 原理：底层是 list，Qt 同步调用，没有异步

Signal is a list — `connect` appends to list, `disconnect` removes from list, `emit` calls all items in list in order. Qt runs slots synchronously (no threading or queue).

### 多槽顺序执行

```python
self.increase = Signal()

self.increase.connect(self.slot_a)  # 按追加顺序加入列表
self.increase.connect(self.slot_b)

def slot_a(self): print('A')
def slot_b(self): print('B')
# 点击时槽按 A → B 顺序执行
```

### 参数匹配规则

`Signal(int, str)` 的槽必须接收两个参数。

```python
from functools import partial

def two_args_handler(n: int, s: str):
    print(n, s)

# lambda 过滤参数（常见写法）
multi_signal.connect(lambda n, s: two_args_handler(n, s))

# partial 补参数（显式写法）
multi_signal.connect(partial(two_args_handler, label='数量'))
```

## `@Slot()` 装饰器

```python
from PySide6.QtCore import Slot

class MyWidget(QWidget):
    @Slot()
    def on_click(self):
        print('clicked')

    @Slot(int)
    def on_value(self, v: int):
        print(f'value: {v}')

    @Slot(int, str)
    def on_multi(self, n: int, s: str):
        print(n, s)
```

装饰器顺序：`@Slot()` 在下，`@Signal()` 在上。

## 注意事项

- `disconnect()` 不加参数调用会清空整个连接列表
- lambda 每 emit 都创建新函数实例，大批量连接时注意性能
- 自定义信号在**类属性位置**定义，不能在 `__init__` 里定义
- `destroyed` 信号：对象销毁时自动发出，适合做清理资源引用
