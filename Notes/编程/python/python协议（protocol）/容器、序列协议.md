# 核心方法

一个容器（如列表、元组、字典、集合等）实现了一下两个核心方法，他就能被python看作是容器对象
- `__len__()`
- `__getitem__(index)`

## 1️⃣`__len__()`
用于告诉python容器的“大小”
>这个方法用于返回容器里的元素数量，容器的长度
>它返回一个整数，表示容器中有多少个元素

```python
class myContainer:
	def __init__(self,items):
		self.items = items
		
	def __len__(self):
		return len(self.items)
```

## 2️⃣`__getitem__(index)`
告诉python如何用索引访问元素
>`__getitem__`是容器中最常用的索引方法，告诉python怎么用索引来获取容器中的某个元素

```python
class myContainer:
	def __init__(self,items):
		self.items = items

	def __len__(self):
		return len(self.items)
		
	def __getitem__(self, index):
		return self.item[index]
```

# 容器协议的实际意义
当一个对象实现了这两个方法，python就认为它是一个容器对象，可以被用于for循环、list（）、sum（）等
- `__len__()`:让对象能使用`len()`函数
- `__getitem__(index)`：让对象用下标访问元素
所以这些方法让对象具备了序列的行为

在实现容器协议后就可以实现迭代协议

# 序列协议
序列协议是容器协议的扩展，不仅包含上面提到的两个方法，还有：
- `__iter__()`：返回迭代器
- `__contains__(item)`：检查容器是否包含某个元素
