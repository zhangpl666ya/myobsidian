> **在 Python 里：  
> 运算符不是语法魔法，  
> 而是“对对象方法的调用”。**

这和C++的运算符重载逻辑上相似
# 从一个最熟的例子开始：`+`

你写过无数次：

`a + b`

Python 实际做的是：

`a.__add__(b)`

如果这一步“不知道怎么加”，  
才会尝试：

`b.__radd__(a)`

例子：
```python
class Vec:
    def __init__(self, x):
        self.x = x

    def __add__(self, other):
        return Vec(self.x + other.x)

```

## `+=` 的规则是：

1. 先看有没有 `__iadd__`
    
2. 没有 → 退化成 `__add__` + 赋值
    

```python
class A:     
	def __init__(self, x):
		self.x = x      
		
	def __iadd__(self, other):         
		self.x += other         
		return self

a = A(1) a += 2 print(a.x)   # 3
```

📌 **`__iadd__` 是“就地修改”协议**

## 为什么要有 `__radd__`？

看这个例子：

`1 + v1`

Python 会做：

1. `int.__add__(1, v1)` → 不认识
    
2. 尝试 `v1.__radd__(1)`
    

所以你可以写：

`class Vec:     def __add__(self, other):         if isinstance(other, Vec):             return Vec(self.x + other.x)         return NotImplemented      def __radd__(self, other):         return Vec(self.x + other)`

📌 **返回 `NotImplemented` 是“礼貌拒绝”**  
不是异常。

## 所有运算符

| 运算符  | 方法             |
| ---- | -------------- |
| `+`  | `__add__`      |
| `-`  | `__sub__`      |
| `*`  | `__mul__`      |
| `/`  | `__truediv__`  |
| `//` | `__floordiv__` |
| `%`  | `__mod__`      |
| `**` | `__pow__`      |
| `-a` | `__neg__`      |

| 表达式      | 调用            |
| -------- | ------------- |
| `a == b` | `a.__eq__(b)` |
| `a != b` | `a.__ne__(b)` |
| `a < b`  | `a.__lt__(b)` |
| `a <= b` | `a.__le__(b)` |
| `a > b`  | `a.__gt__(b)` |
| `a >= b` | `a.__ge__(b)` |
