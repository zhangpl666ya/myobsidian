# Descriptor Portocol

> **描述符 = “类里放了一个特殊对象，这个对象在被当成属性用时，会自动被调用”**

# 前置知识
## 类属性和实例属性
想理解 Python 中**类属性 A.x**和**实例属性 a.x**的区别，核心要抓住一个关键：**A.x 属于类本身，是所有实例共享的；a.x 属于实例 a 自己，是该实例独有的**，二者存储在不同的命名空间中，这是理解差异的根本
```python
class A:
	x = 10 #这是抽象的类的属性
		

a = A()
a.x = 20 # 这是实例属性
print(a.x) #20
print(A.x) #10
```
从结果能明显看到：

- 改`A.x`，所有「没有自己 x」的实例（如 b）都会跟着变；
- 给`a.x`赋值，只是给 a 单独创建了一个 x，和类的 x 彻底脱钩；
- 实例没有自己的属性时，会**向上查找**类的同名属性（这是 Python 的属性查找规则）。
### 底层逻辑：类和实例有各自的「命名空间」

Python 中，**类**和**实例**都是对象，各自有独立的**命名空间**（可以理解为「属性字典」），用来存储自己的属性，这是二者本质不同的原因：

1. 类 A 的命名空间：通过`A.__dict__`查看，`A.x`就存在这里面；
2. 实例 a 的命名空间：通过`a.__dict__`查看，只有给`a.x`赋值后，`a.__dict__`里才会有 x，否则为空。
3. 为了区别，实例属性前面通常会有"_ “如”_x "

python的类有一个字典：`__dict__`
这是一个字典存储这个类拥有的属性和值
```python
class A:
	x = 10

a = A()
print(A.__dict__)#'x':10
print(a.__dict__)#空的
a.x = 20
print(a.__dict__)#'x': 20
```

# 描述符
类中定义了这些函数任意一种就是描述符：
`__get__ __set__ __delete__`
分别对应：属性的访问、设置、删除
我们可以理解为将访问、设置、删除包装为函数
```python
class Field:
	def __get__(self,instance,owner):
		print('visit')
		return instance.__dict__['_x']
		
	def __set__(self,instance,value):
		print('set')
		instance.__dict__['_x'] = value
		
	def __delete__(self,instance):
		print('delete')
		instance.__dict__.pop('_x',None)

class A:
	x = Field()
```
这个例子中将A的属性x的描述符完善，及可以理解为x这个属性实现了描述符协议。当我们访问、设置、删除x的时候就会对应调用这些函数
- `__get__`
	- 第一个参数是本身
	- 第二个参数是实例
	- 第三个参数是类
- `__set__`
	- 第一个参数是自身
	- 第二个参数是实例
	- 第三个参数是要改成的值
- `delete`
	- 同上

Python 规定，只要一个对象实现了以下三个特殊方法（协议方法）中的至少一个，它就是「描述符对象」：

|协议方法|作用|触发时机|参数说明|
|---|---|---|---|
|`__get__(self, instance, owner)`|处理「读取属性」的逻辑|`obj.attr` 或 `Class.attr`|self = 描述符对象自身；instance = 访问属性的实例（如 a）；owner = 所属的类（如 A）|
|`__set__(self, instance, value)`|处理「修改属性」的逻辑|`obj.attr = value`|self = 描述符对象；instance = 实例；value = 要设置的新值|
|`__delete__(self, instance)`|处理「删除属性」的逻辑|`del obj.attr`|self = 描述符对象；instance = 实例|

# 与属性访问协议的关系

述符的所有方法，都是在属性访问协议的执行过程中被调用的

最典型的例子：`obj.foo`的执行流程（Python 的默认逻辑），这一步同时触发了**属性访问协议**和（如果`foo`是描述符的话）**描述符协议**：

1. 执行`obj.__getattribute__('foo')`（属性访问协议的核心，**所有属性访问的第一步**）；
2. `__getattribute__`内部按规则查找`foo`：先找实例的`__dict__`，再找类的`__dict__`，再找父类的`__dict__`；
3. 若找到的`foo`是**描述符对象**（实现了`__get__`/`__set__`/`__delete__`），则**自动调用**`foo.__get__(obj, obj.__class__)`（描述符协议），并将返回值作为`obj.foo`的结果；
4. 若找到的`foo`不是描述符，则直接返回`foo`本身；
5. 若全程没找到`foo`，则触发`obj.__getattr__('foo')`（属性访问协议的兜底）。

简单说：**描述符是属性访问协议的 “插件”**，Python 的属性访问协议预留了扩展口，让符合描述符协议的对象能接管属性的操作，这就是两者重叠的核心。
# 现代描述符
## `property`

>就是官方帮你写的Field类

```python
class A:
    def get_x(self):
        return self._x

    def set_x(self, value):
        self._x = value

    x = property(get_x, set_x)

```
将函数按照：`__get__ __set__ __delete__`顺序传入即可
注意操作的属性要在前面加下划线

## 装饰器
装饰器就大概有：`@property @.setter @.deleter`
分别对应访问、修改、删除
```python
class A:
	@property #装饰器符号
	def x(self):
		print('visit')
		return self._x
	
	@x.setter
	def x(self,value):
		print('set')
		self._x = value
	@x.deleter
	def x(self):
		print('delete')
		del self._x
```

用一张表总结装饰器对属性查找的影响，结合最初的`A.x`和`a.x`：

|操作|原生逻辑（无装饰器）|装饰后逻辑（有 @property / 类装饰器）|
|---|---|---|
|`a.x`|先找 a.__dict__的 x，无则找 A.__dict__的 x|先找 a.__dict__的 x（有则用），无则找 A 的 x；若 A 的 x 是受控属性，触发类属性的 get 逻辑|
|`a.x = v`|直接在 a.__dict__中创建 / 修改 x（实例属性）|若 a 有受控的实例属性 x，触发 set 逻辑（校验）；无则直接创建实例属性 x|
|`A.x`|直接找 A.__dict__的 x（类属性）|若 A 的 x 是受控类属性，触发类属性的 get/set 逻辑（校验）|
|`del a.x`|直接删除 a.__dict__的 x|若 a 有受控的实例属性 x，触发 deleter 逻辑；无则报错|

### 装饰器扩展
`@staticmethod`（静态方法装饰器）和`@classmethod`（类方法装饰器）的核心区别与使用场景，核心要抓住：**二者都是基于「非数据描述符」实现的，核心差异在于「绑定对象不同」—— 静态方法不绑定任何对象，类方法绑定类本身**，这也是它们和普通实例方法的根本区别。

先用对比表理清`@staticmethod`、`@classmethod`和普通实例方法的关键差异，一目了然：

|特性|普通实例方法|`@classmethod`（类方法）|`@staticmethod`（静态方法）|
|---|---|---|---|
|绑定对象|实例（`self`）|类（`cls`）|无（不绑定）|
|第一个参数|必须是`self`（实例）|必须是`cls`（类）|无默认参数（和普通函数一样）|
|访问属性能力|可访问实例属性 / 类属性|可访问类属性（不能访问实例属性）|不能直接访问实例 / 类属性（需手动传）|
|调用方式|实例调用（`a.func()`）|类调用（`A.func()`）/ 实例调用（`a.func()`）|类调用 / 实例调用（和类方法一样）|
|底层实现|非数据描述符|非数据描述符|非数据描述符|
|核心用途|操作实例的状态|操作类的状态 / 创建实例|纯工具函数（和类 / 实例无关）|
