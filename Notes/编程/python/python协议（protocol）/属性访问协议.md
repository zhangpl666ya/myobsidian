**python是如何读取/设置/找不到属性的？**

属性访问协议
>只要实现一下方法之一
>就参与了python的属性访问流程


- `__getattribute__(self, name)
- `__getattr__(self, name)
- `__setattr__(self, name, value)
- `__delattr__(self, name)
⚠️ 这四个不是并列难度，
`__getattribute__` 是最底层、最危险的那个。

# python底层逻辑理解
从C++的角度解释python。
python的一切都是对象，都是类的实例
而这些类有一个最底层的父类型：`object`
所有类都是继承自它，它实现了很多魔法方法、方法
当我们在自己写class时，若没有写魔法方法，那么默认调用`object`对应的魔法方法

# 1️⃣`__getattr__`
什么时候被调用？
👉只有当**正常属性查找失败时才调用**，若存在就不会
```python
class A:
	def __getattr__(self, name):
		return f"<no such attribute:{name}>"
		
a = A()
print(a.x) #no such attribute:x
```

## 属性查找的“完整流程”（极其重要）

当你访问：

`obj.attr`

Python 实际做的是：

1. 调用 `obj.__getattribute__("attr")`
	
2. 查看类字典
    
3. 如果找到了 → 返回
    
4. 如果抛 `AttributeError`  
    → 再调用 `__getattr__`
    

👉 **`__getattr__` 是兜底，不是入口**

# 2️⃣`__getattribute__`
所有属性访问都会经过它，不管你访问的属性是否存在
这是调用属性的总入口，若不存在，则会抛出错误，执行`__getattr__()`
```python
class A:
	def __getattribute__(self, name):
		print("访问属性",name)
		return object.__getattribute__(self,name)
```

>[!warning]
>这里返回的必须是`object`的魔法方法，若还调用自己的会陷入死循环

# 3️⃣`__setattr__`
每次给属性赋值的时候就会调用
```python
class A:
    def __setattr__(self, name, value):
        print(f"我要设置 {name} = {value}")
        object.__setattr__(self, name, value)

a = A()
a.x = 10
#输出： 我要设置x = 10

```
# 4️⃣`__delattr__`
用于删除这个属性
```python
class A:
	def  __delattr__(self, name):
		print('删除：{name}')
		object.__delattr__(self,name)
a.x = 10
del a.x
#输出：删除：x
```
