# 可调用协议（Callable Protocol)
>只要一个对象实现了`__call__`，就可以像函数一样调用

当该对象实现这个魔法方法且使用括号调用时，就相当于调用这个魔法方法

```python
class Adder:
	def __init__(self, n):
		self.n = n
	
	def __call__(self,x):
		return self.n + x
		
add5 = Adder(5)
print(add5(10)) #15
```

实际是`add5(10) -> add5.__call__(10)`

# Python 眼里的“函数”到底是什么？

这是理解可调用协议的关键。

### 在 Python 里：

> **函数 ≈ 实现了 `__call__` 的对象**

- 普通函数对象：有 `__call__`
    
- lambda：有 `__call__`
    
- 类实例：只要你写了 `__call__`，它也“像函数”
    

📌 Python **不区分“函数”和“可调用对象”**

# 可调用 ≠ 函数（这是关键区分）

|对比项|函数|可调用对象|
|---|---|---|
|是否有状态|❌|✅|
|是否是对象|✅|✅|
|是否能 `()`|✅|✅|
|是否能保存配置|❌|✅|
# 应用：装饰器

**装饰器是什么**？
>装饰器 = 一个“接收函数 -> 返回新函数（可调用对象）”的东西

形式上：`decorator(func) -> new_func`

```python
def add(a,b):
	return a+b

#装饰它
def logger(func):
	def wrapper(*args,**kwargs):
		return func(*args, **kwargs)
	return wrapper
```
