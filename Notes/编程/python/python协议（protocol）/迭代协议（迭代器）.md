## 一、先分清两个概念（非常重要）

很多人混在一起，其实 Python **明确区分了两种东西**：

### 1️⃣ 可迭代对象（Iterable）

- 能被 `for` 循环
    
- 能被 `iter(obj)` 调用
    
- **实现了 `__iter__`**
    

例子：

```python
list
tuple
str
dict
set
```

### 2️⃣ 迭代器（Iterator）

- 真正“一步一步吐数据”的对象
    
- **实现了**
    
    - `__iter__`
        
    - `__next__`
        

📌 **所有迭代器都是可迭代对象，但反过来不成立**

---

## 二、`__iter__` 是干嘛的？

### 作用一句话：

> **返回一个“迭代器”**

### 也就是说：

```python
iter(obj)   ⟶   obj.__iter__()
```

### 例子（概念版）：

```python
it = iter([1, 2, 3])
```

这里：

- `[1,2,3]` 是 **可迭代对象**
    
- `it` 是 **迭代器**
    

>[!danger]
>注意：自定义类的`__iter__(self)`最好返回一个独立的迭代对象，不要返回自己，这样的话只能迭代一次！

```python

```

---

### 🔍 关键点（非常容易被忽略）

`__iter__` **不一定返回自己**

- list 的 `__iter__` → 返回一个 **新的 list_iterator**
    
- 文件对象的 `__iter__` → 返回自己
    
- 你自己写的类 → 想返回啥都行（只要是迭代器）
    

---

## 三、`__next__` 是干嘛的？

### 作用一句话：

> **返回“下一个值”，没有了就抛 `StopIteration`**

### 本质：

```python
next(it)   ⟶   it.__next__()
```

### 规则是铁律：

- 有下一个值 → `return value`
    
- 没有值了 → **必须抛 `StopIteration`**
    

❌ 不能返回 `None` 表示结束  
❌ 不能用别的异常

---

## 四、`for` 循环到底干了什么？（核心）

你写的：

```python
for x in obj:
    print(x)
```

**等价于：**

```python
it = iter(obj)      # 调 __iter__

while True:
    try:
        x = next(it)   # 调 __next__
    except StopIteration:
        break
    print(x)
```

🔥 所以你现在能看到：

- `__iter__`：只调用 **一次**
    
- `__next__`：调用 **很多次**

## 五、一个完整的自定义例子

```python
class MyRange:
    def __init__(self, n):
        self.n = n
        self.i = 0

    def __iter__(self):
        return self    # 我自己就是迭代器

    def __next__(self):
        if self.i >= self.n:
            raise StopIteration
        value = self.i
        self.i += 1
        return value
```

使用：

```python
for x in MyRange(3):
    print(x)
```

输出：

```
0
1
2
```

## 六、为什么要分 `__iter__` 和 `__next__`？

这是**设计上的深思熟虑**，不是多此一举。

### 1️⃣ 一个容器可以产生多个迭代器

```python
lst = [1, 2, 3]

it1 = iter(lst)
it2 = iter(lst)

next(it1)  # 1
next(it2)  # 1
```

➡️ 如果只有 `__next__`，就做不到这一点

---

### 2️⃣ 支持惰性计算（lazy evaluation）

- 文件逐行读
    
- 生成器
    
- 无限序列
    

这些都**不能一次性算完**

---

## 七、生成器是怎么回事？

```python
def gen():
    yield 1
    yield 2
```

其实：

- `gen()` 返回的是 **一个迭代器**
    
- Python 自动帮你实现了：
    
    - `__iter__`
        
    - `__next__`
        
    - `StopIteration`
        

👉 **生成器 = 自动帮你写好的迭代器类**

---

## 八、一句总结（记住这个就行）

> - `__iter__`：**告诉 Python 从哪拿迭代器**
>     
> - `__next__`：**告诉 Python 下一步给什么**
>     
> - `for`：**反复调用 `next()`，直到 `StopIteration`**


> **`__iter__` 决定“我能不能被迭代”**  
> **`__next__` 决定“下一步拿什么”**

它们一起组成了 **Python 的迭代器协议（iterator protocol）**。

---
## Python 在哪用这个协议？

### `for`

`for x in obj:     ...`

### `list()`

`list(obj)`

### `sum()`

`sum(obj)`

### 解包

`a, b, c = obj`

### 推导式

`[x for x in obj]`

👉 **只要是“遍历”，底层一定走迭代协议**
